# 📖 Ситуация

В ресторане **DineFlow** используется настольное приложение для регистрации и обработки бронирований столиков.

Приложение позволяет хостес и администраторам:

* создавать новые бронирования для гостей (внутренние, терраса, VIP);
* отслеживать статус их подтверждения;
* управлять жизненным циклом бронирований через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать имена гостей с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации бронирования

Форма используется для создания нового бронирования столика.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Guest

Назначение:

ввод имени гостя.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Host

Назначение:

имя сотрудника (хоста), принимающего бронирование.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Table Type

Назначение:

выбор типа столика.

Возможные значения:

* Indoor
* Outdoor
* VIP

---

## Список Priority

Назначение:

выбор приоритета бронирования.

Возможные значения:

* Low
* Medium
* High

---

# Таблица бронирований

Структура таблицы:

| ID | Guest | Host | Table Type | Priority | Status |

---

# Возможные статусы

```
New
Confirmed
```

---

# Кнопки управления бронированиями

## Assign Host

Назначает ответственного хоста на выбранное бронирование.

---

## Confirm Booking

Изменяет статус выбранного бронирования на

```
Confirmed
```

Кнопка неактивна, если статус бронирования уже `"Confirmed"` или строка не выбрана.

# Шаг 2. Генерация имён гостей через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно сгенерировать тестовые имена гостей**, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 вымышленных имён гостей для ресторана DineFlow. Используй разнообразные имена. Верни только список имён без номеров, каждое имя с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |
