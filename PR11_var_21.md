# 📖 Ситуация

В логистической компании **ShipFlow** используется настольное приложение для регистрации и обработки отгрузок товаров.

Приложение позволяет сотрудникам склада и логистам:

* создавать новые записи об отгрузках (локальных, региональных, международных);
* отслеживать статус их выполнения;
* управлять жизненным циклом отгрузок через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать наименования грузов с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации отгрузки

Форма используется для создания новой записи об отгрузке.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Item

Назначение:

ввод наименования груза или товара.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Operator

Назначение:

имя оператора, ответственного за отгрузку.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Shipment Type

Назначение:

выбор типа отгрузки.

Возможные значения:

* Local
* Regional
* International

---

## Список Priority

Назначение:

выбор приоритета обработки.

Возможные значения:

* Low
* Medium
* High

---

# Таблица отгрузок

Структура таблицы:

| ID | Item | Operator | Shipment Type | Priority | Status |

# Возможные статусы

```

Shipped
```

# Кнопки управления отгрузками

## Assign Operator

Назначает ответственного оператора на выбранную отгрузку.

Кнопка активна только при выборе строки в таблице.

---

## Ship Item

Изменяет статус выбранной отгрузки на

```
Shipped
```

Кнопка неактивна, если статус отгрузки уже `"Shipped"` или строка не выбрана.

# Шаг 2. Генерация наименований грузов через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать названия грузов** для разных категорий, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 наименований грузов для логистической системы. Темы: электроника, мебель, медикаменты, офисные принадлежности, розничные товары, запчасти, продукты питания, текстиль, химикаты, автозапчасти. Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |

---
