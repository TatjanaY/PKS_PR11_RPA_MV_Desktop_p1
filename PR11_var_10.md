# 📖 Ситуация

В компании **HRFlow** используется настольное приложение для регистрации и обработки кадровых заявок сотрудников.

Приложение позволяет специалистам HR-отдела и руководителям:

* создавать новые заявки на отпуск, больничный или командировку;
* отслеживать статус их согласования;
* управлять жизненным циклом заявок через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать темы HR-заявок с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации заявки

Форма используется для создания новой кадровой заявки.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Request Title

Назначение:

ввод темы или краткого описания заявки.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Employee

Назначение:

имя сотрудника, подающего заявку.

Примеры:

* Alexey Volkov
* Marina Sokolova

---

## Поле HR Manager

Назначение:

имя HR-менеджера, ответственного за обработку.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Request Type

Назначение:

выбор типа кадрового запроса.

Возможные значения:

* Vacation
* Sick Leave
* Business Trip

---

## Список Priority

Назначение:

выбор приоритета обработки заявки.

Возможные значения:

* Low
* Medium
* High

---

# Шаг 2. Генерация тем HR-заявок через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать темы кадровых заявок**, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 тем HR-заявок для корпоративного портала. Темы: оформление ежегодного отпуска, предоставление больничного листа, согласование командировки в Москву, запрос на удалённую работу, оформление декретного отпуска, замена оборудования на рабочем месте, запрос на обучение, пересмотр заработной платы, перевод в другой отдел, оформление донорского дня. Верни только список тем без номеров, каждая тема с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |
