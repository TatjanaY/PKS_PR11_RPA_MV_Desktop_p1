# 📖 Ситуация

В автосервисе **AutoFlow** используется настольное приложение для регистрации и обработки заявок на ремонт автомобилей.

Приложение позволяет сотрудникам сервиса и мастерам-приёмщикам:

* создавать новые заявки на ремонт (двигатель, кузов, электрика);
* отслеживать статус их выполнения;
* управлять жизненным циклом заявок через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать модели автомобилей и описания неисправностей с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации заявки

Форма используется для создания новой заявки на ремонт автомобиля.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Car

Назначение:

ввод модели автомобиля и краткого описания неисправности.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Mechanic

Назначение:

имя механика, ответственного за заявку.

* Ivan Petrov
* Anna Smirnova

## Список Service Type

Назначение:

выбор типа сервиса/ремонта.

Возможные значения:

* Engine
* Body
* Electrical

---

## Список Priority

Назначение:

выбор приоритета обработки заявки.

Возможные значения:

* Low
* Medium
* High

# Шаг 2. Генерация автомобилей и неисправностей через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать модели автомобилей и описания проблем**, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 описаний автомобилей с указанием неисправности для автосервиса AutoFlow. Темы: стук в двигателе, замена тормозных колодок, ремонт генератора, диагностика подвески, замена масла, покраска крыла, ремонт стартера, замена аккумулятора, сход-развал, ремонт рулевой рейки. Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |
