# 📖 Ситуация

В компании **WarehouseFlow** используется настольное приложение для регистрации и обработки складских операций.

Приложение позволяет сотрудникам логистического отдела и операторам склада:

* создавать новые записи о приёмке, отгрузке и перемещении товаров;
* отслеживать статус выполнения операций;
* управлять жизненным циклом записей через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать наименования товаров для складских операций с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации операции

Форма используется для создания новой записи о перемещении или приёмке/отгрузке товара.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Item

Назначение:

ввод наименования товара или оборудования.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Operator

Назначение:

имя оператора, оформившего операцию.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Operation Type

Назначение:

выбор типа складской операции.

Возможные значения:

* Inbound
* Outbound
* Transfer

---

## Список Priority

Назначение:

выбор приоритета обработки операции.

Возможные значения:

* Low
* Medium
* High

# Шаг 2. Генерация наименований товаров через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать названия товаров** для складских операций, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 наименований товаров или оборудования для складского учёта. Темы: ноутбуки, мониторы, периферия (мыши, клавиатуры), кабели, сетевое оборудование, зарядные устройства, веб-камеры, док-станции. Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |
