# 📖 Ситуация

В городской библиотеке **BookFlow** используется настольное приложение для регистрации и обработки запросов читателей на получение книг.

Приложение позволяет сотрудникам библиотеки:

* создавать новые запросы на выдачу, продление или бронирование книг;
* отслеживать статус их выполнения;
* управлять жизненным циклом запросов через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать названия книг и темы запросов с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации запроса

Форма используется для создания нового запроса читателя.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Book Title

Назначение:

ввод названия книги.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Reader

Назначение:

имя читателя, оформляющего запрос.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Category

Назначение:

выбор категории книги.

Возможные значения:

* Fiction
* Science
* History

---

## Список Request Type

Назначение:

выбор типа запроса.

Возможные значения:

* Borrow
* Extend
* Reserve

# Кнопки управления запросами

## Mark as Issued

Изменяет статус выбранной заявки на

```

```

Кнопка неактивна, если статус заявки уже `"Issued"` или строка не выбрана.

## Assign Librarian

Назначает библиотекаря на выбранный запрос.

Кнопка активна только при выборе строки в таблице.

# Шаг 2. Генерация названий книг через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать названия книг** для разных категорий, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значание                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 названий книг для библиотеки. Распредели по категориям: 4 Fiction (романы, детективы), 3 Science (научпоп, учебники), 3 History (исторические исследования, биографии). Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |

---
