# 📖 Ситуация

В компании **TaskFlow** используется настольное приложение для регистрации и отслеживания рабочих задач сотрудников.

Приложение позволяет руководителям и членам команд:

* создавать новые задачи по разработке, тестированию и документации;
* отслеживать статус их выполнения;
* управлять жизненным циклом задач через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать наименования задач с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации задачи

Форма используется для создания новой рабочей задачи.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Task

Назначение:

ввод наименования или описания задачи.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Assignee

Назначение:

имя исполнителя, ответственного за задачу.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Category

Назначение:

выбор категории задачи.

Возможные значения:

* Development
* Testing
* Documentation

---

## Список Priority

Назначение:

выбор приоритета обработки задачи.

Возможные значения:

* Low
* Medium
* High

---

# Таблица задач

Структура таблицы:

| ID | Task | Assignee | Category | Priority | Status |

# Кнопки управления задачами

# Шаг 2. Генерация наименований задач через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать названия задач** для разных категорий, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 наименований задач для IT-команды TaskFlow. Распредели по категориям: 4 Development (аутентификация, баг-фиксы, оптимизация БД, аудит безопасности), 3 Testing (юнит-тесты, интеграционные тесты, тестирование мобильного приложения), 3 Documentation (API-документация, руководство пользователя, заметки к релизу). Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |
