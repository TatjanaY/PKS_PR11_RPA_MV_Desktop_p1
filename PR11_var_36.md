# 📖 Ситуация

В компании **RequestFlow** используется настольное приложение для регистрации и обработки общих корпоративных заявок.

Приложение позволяет сотрудникам и специалистам:

* создавать новые заявки на технические, административные и финансовые запросы;
* отслеживать статус их выполнения;
* управлять жизненным циклом заявок через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать описания общих заявок с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации заявки

Форма используется для создания нового корпоративного запроса.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Request

Назначение:

ввод описания или темы заявки.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Specialist

Назначение:

имя специалиста, ответственного за обработку.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Request Type

Назначение:

выбор типа заявки.

Возможные значения:

* Technical
* Administrative
* Financial

---

## Список Priority

Назначение:

выбор приоритета обработки заявки.

Возможные значения:

* Low
* Medium
* High

---

# Кнопки управления заявками

# Шаг 2. Генерация описаний заявок через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать описания корпоративных заявок**, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 описаний общих корпоративных заявок для системы RequestFlow. Темы: обновление серверов, заказ канцелярии, согласование бюджета, настройка сети, командировочные расходы, лицензии на ПО, обновление межсетевого экрана, обработка счетов, бронирование переговорной, оптимизация базы данных. Верни только список описаний без номеров, каждое описание с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |
