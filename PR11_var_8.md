# 📖 Ситуация

В клинике **MedFlow** используется настольное приложение для регистрации и обработки медицинских записей и приёмов пациентов.

Приложение позволяет сотрудникам регистратуры и врачам:

* создавать новые записи на приём, диагностику и процедуры;
* отслеживать статус подтверждённых приёмов;
* управлять жизненным циклом записей через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать темы медицинских приёмов с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# 🖥 Используемое приложение

Приложение состоит из:

* формы регистрации медицинской записи;
* таблицы зарегистрированных приёмов;
* кнопок управления записями.

---

# Форма регистрации записи

Форма используется для создания новой записи пациента на приём или процедуру.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Appointment Topic

Назначение:

ввод темы приёма или процедуры.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Patient

Назначение:

имя пациента, записанного на приём.

---

## Поле Doctor

Назначение:

имя врача, проводящего приём.

---

## Список Department

Назначение:

выбор отделения клиники.

Возможные значения:

* Cardiology
* Neurology
* Therapy

---

## Список Priority

Назначение:

выбор приоритета обработки записи.

Возможные значения:

* Low
* Medium
* High

---

# Таблица записей

Структура таблицы:

| ID | Patient | Doctor | Department | Priority | Status |

---

# Возможные статусы

```
New
Confirmed
```

# Кнопки управления записями

## Assign Nurse

Назначает ответственную медсестру на выбранную запись.

Кнопка активна только при выборе строки в таблице.

---

## Confirm Appointment

Изменяет статус выбранной записи на

```
Confirmed
```

Кнопка неактивна, если статус записи уже `"Confirmed"` или строка не выбрана.

# Шаг 2. Генерация тем приёмов через GPT

Это демонстрирует интеграцию **LLM в RPA-процесс**.

Использовать блок:

**AI → Выполнить запрос в GPT**
