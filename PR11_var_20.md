# 📖 Ситуация

В компании **ProjectFlow** используется настольное приложение для регистрации и обработки корпоративных проектов.

Приложение позволяет руководителям и менеджерам:

* создавать новые проекты (внутренние, внешние, исследовательские);
* отслеживать статус их запуска;
* управлять жизненным циклом проектов через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать названия проектов с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Шаг 2. Генерация названий проектов через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать названия проектов** для разных типов, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 названий корпоративных проектов для системы ProjectFlow. Распредели по типам: 4 Internal (оптимизация процессов, обновление CRM, миграция данных, внутренний аудит), 3 External (клиентский портал, интеграция с партнёрами, запуск мобильного приложения), 3 Research (ML-модели, AI-ассистент, анализ больших данных). Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |

---

```
Оптимизация внутренней документооборота

Проведение внутреннего аудита безопасности

```
