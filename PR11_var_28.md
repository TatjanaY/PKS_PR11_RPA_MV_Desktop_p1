# 📖 Ситуация

В компании **PayFlow** используется настольное приложение для регистрации и обработки клиентских платежей.

Приложение позволяет сотрудникам:

* создавать новые записи о платежах;
* отслеживать их статус;
* управлять обработкой платежей через таблицу.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы;
* внутри виртуальных рабочих сред;
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их изображению;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать описания платежей с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Шаг 2. Генерация описаний платежей через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать осмысленные описания для 10 платежей**, а не брать их из файла. Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значание                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT (или ChatGPT / YandexGPT)                                                                                                                                                                                                                                                                  |
| Запрос                     | Сгенерируй 10 кратких описаний платежей для финансовой системы. Темы: подписки, комиссии, возвраты, апгрейды, лицензии, поддержка, разовые платежи, кастомные решения. Верни только список описаний без номеров и лишнего текста, каждое описание с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |

### Результат

Блок вернёт строку с 10 описаниями платежей, разделёнными переводами строки.

---

## Пример ответа GPT

```
Оплата ежемесячной подписки CRM
Возврат средств за неиспользованный тариф
Обновление лицензии до Enterprise
Оплата пакета технической поддержки
Единоразовый платеж за консультацию
Списание комиссии за обработку перевода
Оплата корпоративного облачного хранилища
Взнос за кастомную интеграцию API
Платеж за ежегодный аудит безопасности
Оплата расширения пользовательских прав
```

# Шаг 3. Преобразование ответа GPT в массив

## Зачем выполняется шаг

---

## Что необходимо сделать

Использовать операцию:

**Разделение строки по символу перевода строки**

Результат:

массив из **10 описаний платежей** для последующего использования в цикле.
