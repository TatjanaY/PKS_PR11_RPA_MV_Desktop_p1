# 📖 Ситуация

В компании **IncidentFlow** используется настольное приложение для регистрации и обработки инцидентов информационной безопасности и системных сбоев.

Приложение позволяет сотрудникам SOC и системным администраторам:

* создавать новые записи об инцидентах (Security, System, Network);
* отслеживать статус их разрешения;
* управлять жизненным циклом инцидентов через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать описания инцидентов с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации инцидента

Форма используется для создания новой записи об инциденте.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Incident

Назначение:

ввод описания или названия инцидента.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Operator

Назначение:

имя оператора, зарегистрировавшего инцидент.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Incident Type

Назначение:

выбор типа инцидента.

Возможные значения:

* Security
* System
* Network

---

## Список Priority

Назначение:

выбор приоритета обработки инцидента.

Возможные значения:

* Low
* Medium
* High

---

# Шаг 2. Генерация описаний инцидентов через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать описания инцидентов** для разных типов, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 кратких описаний инцидентов для системы мониторинга IncidentFlow. Распредели по типам: 4 Security (несанкционированный доступ, утечка данных, подозрительная активность, истёкший сертификат), 3 System (падение сервера, блокировка БД, ошибка приложения), 3 Network (задержка сети, сбой DNS, отказ маршрутизатора). Верни только список описаний без номеров, каждое описание с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |

---
