# 📖 Ситуация

В компании **SupportFlow** используется настольное приложение для регистрации и обработки обращений клиентов.

Приложение позволяет операторам поддержки:

* создавать новые обращения (жалобы, запросы, обратную связь);
* отслеживать статус их обработки;
* управлять жизненным циклом обращений через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать имена клиентов для тестовых обращений с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации обращения

Форма используется для создания нового обращения клиента.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Client

Назначение:

ввод имени клиента.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Operator

Назначение:

имя оператора, оформившего обращение.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Request Category

Назначение:

выбор категории запроса.

Возможные значения:

* Complaint
* Inquiry
* Feedback

---

## Список Priority

Назначение:

выбор приоритета обработки обращения.

Возможные значения:

* Low
* Medium
* High

---

# Таблица обращений

Структура таблицы:

| ID | Client | Operator | Request Category | Priority | Status |

# Возможные статусы

```

# Кнопки управления обращениями

Назначает ответственного оператора на выбранное обращение.

```

Кнопка неактивна, если статус обращения уже `"Closed"` или строка не выбрана.

# Шаг 2. Генерация имен клиентов через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать имена клиентов** для тестовых обращений, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 вымышленных имен клиентов (фамилия и имя) для системы SupportFlow. Используй разнообразные российские имена. Верни только список имен без номеров, каждое имя с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |
