# 📖 Ситуация

В компании **SupportFlow** используется настольное приложение для регистрации и обработки тикетов технической поддержки.

Приложение позволяет сотрудникам службы поддержки и агентам:

* создавать новые тикеты (баги, вопросы, запросы на функционал);
* отслеживать статус их решения;
* управлять жизненным циклом обращений через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать описания тикетов с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации тикета

Форма используется для создания нового обращения в службу поддержки.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Ticket

Назначение:

ввод описания проблемы или запроса.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Support Agent

Назначение:

имя агента поддержки, регистрирующего тикет.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Issue Type

Назначение:

выбор типа проблемы.

Возможные значения:

* Bug
* Question
* Request

---

## Список Priority

Назначение:

выбор приоритета обработки тикета.

Возможные значения:

* Low
* Medium
* High

# Кнопки управления тикетами

## Assign Support Agent

# Шаг 2. Генерация описаний тикетов через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать описания тикетов** для разных типов обращений, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 описаний тикетов для службы поддержки SupportFlow. Темы: ошибка авторизации, вопрос по оплате, запрос на новую функцию, сбой в работе приложения, проблема с экспортом отчётов, запрос на сброс пароля, баг в мобильной версии, вопрос по интеграции API, запрос на удаление аккаунта, ошибка загрузки файлов. Верни только список описаний без номеров, каждое описание с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |

---
