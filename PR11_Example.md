# Автоматизация десктоп-приложения с использованием Machine Vision в Puzzle RPA

---

# 📖 Ситуация

В компании **TechFlow** используется настольное приложение для регистрации и обработки сервисных заявок сотрудников.

Приложение позволяет сотрудникам:

* создавать новые заявки;
* отслеживать их статус;
* управлять обработкой заявок через таблицу.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы;
* внутри виртуальных рабочих сред;
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса, основанная на **UI-селекторах**, может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их изображению;
* выполняет действия по координатам найденных элементов.

В рамках данной практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который взаимодействует с приложением **исключительно через поиск изображений элементов интерфейса**.

---

# 🖥 Используемое приложение

Автоматизация выполняется для настольного приложения регистрации заявок.

Приложение состоит из:

* формы создания заявки;
* таблицы заявок;
* кнопок управления записями.

# Форма регистрации заявки

Форма используется для создания новой сервисной заявки.

Пользователь заполняет поля формы и добавляет запись в таблицу.

---

## Поле Request Title

Назначение элемента:

ввод названия или темы заявки.

Примеры:

* Access to CRM
* Setup workstation
* VPN connection

Робот вводит значение из Excel-файла.

---

## Поле Employee

Назначение элемента:

имя сотрудника, создающего заявку.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Priority

Назначение элемента:

выбор приоритета обработки заявки.

Возможные значения:

* Low
* Medium
* High
