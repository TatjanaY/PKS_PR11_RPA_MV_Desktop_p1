# 📖 Ситуация

В компании **EventFlow** используется настольное приложение для регистрации и корпоративных мероприятий (конференций, вебинаров, митапов и других событий).

Приложение позволяет организаторам и координаторам:

* создавать новые события с указанием типа и приоритета;
* отслеживать статус их подготовки и запуска;
* управлять жизненным циклом мероприятий через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать названия мероприятий с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# 🖥 Используемое приложение

Приложение состоит из:

* формы регистрации события;
* таблицы зарегистрированных мероприятий;

---

# Форма регистрации события

Форма используется для создания новой записи о мероприятии.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Event

Назначение:

ввод названия или темы мероприятия.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Coordinator

Назначение:

имя координатора, ответственного за мероприятие.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Event Type

Назначение:

выбор типа мероприятия.

Возможные значения:

* Conference
* Webinar
* Meetup

---

## Список Priority

Назначение:

выбор приоритета подготовки события.

Возможные значения:

* Low
* Medium
* High

---

# Таблица событий

Структура таблицы:

| ID | Event | Coordinator | Event Type | Priority | Status |

---

# Возможные статусы

```
New
Started
```

---

# Кнопки управления событиями

## Assign Coordinator

Назначает ответственного координатора на выбранное событие.

---

## Start Event

Изменяет статус выбранного события на

```
Started
```

Кнопка неактивна, если статус события уже `"Started"` или строка не выбрана.

# Шаг 2. Генерация названий мероприятий через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать названия мероприятий** для разных типов событий, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 названий корпоративных мероприятий для системы EventFlow. Распредели по типам: 4 Conference (технологические конференции, бизнес-саммиты, отраслевые форумы), 3 Webinar (онлайн-мастерклассы, презентации продуктов, обучающие сессии), 3 Meetup (встречи разработчиков, нетворкинг, хакатоны). Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |
