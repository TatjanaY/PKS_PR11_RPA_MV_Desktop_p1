# 📖 Ситуация

В компании **MaintainFlow** используется настольное приложение для регистрации и обработки задач технического обслуживания оборудования.

Приложение позволяет инженерам и диспетчерам:

* создавать новые задачи по профилактике, ремонту и инспекции;
* отслеживать статус выполнения работ;
* управлять жизненным циклом задач через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать наименования оборудования и описания задач техобслуживания с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

В данной работе значение **генерируется автоматически с помощью GPT**.

# Шаг 2. Генерация наименований оборудования через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать наименования оборудования и задачи** для разных типов обслуживания, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 наименований оборудования с указанием типа задачи техобслуживания для компании MaintainFlow. Распредели по типам: 4 Preventive (замена фильтров, проверка уровней масла, очистка систем охлаждения, калибровка датчиков), 3 Repair (устранение утечки, замена подшипника, ремонт насоса), 3 Inspection (проверка пожарной сигнализации, тестирование генератора, аудит электрощита). Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |

---
