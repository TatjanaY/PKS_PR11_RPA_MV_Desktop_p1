# 📖 Ситуация

В компании **RoomFlow** используется настольное приложение для регистрации и обработки бронирований переговорных комнат.

Приложение позволяет сотрудникам и координаторам:

* создавать новые бронирования для встреч, совещаний и презентаций;
* отслеживать статус подтверждённых бронирований;
* управлять жизненным циклом бронирований через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать названия мероприятий и темы встреч с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации бронирования

Форма используется для создания нового бронирования переговорной комнаты.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Room

Назначение:

ввод названия комнаты или темы мероприятия.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Employee

Назначение:

имя сотрудника, оформившего бронирование.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Location

Назначение:

выбор расположения комнаты.

Возможные значения:

* Floor1
* Floor2
* Floor3

---

## Список Priority

Назначение:

выбор приоритета бронирования.

Возможные значения:

* Low
* Medium
* High

# Шаг 2. Генерация названий мероприятий через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать названия встреч и мероприятий**, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 названий корпоративных мероприятий для бронирования переговорных. Темы: стратегическое планирование, онбординг новых сотрудников, презентация продукта, обучение по безопасности, командная встреча, интервью с кандидатом, воркшоп по инновациям, квартальный отчёт, мозговой штурм, встреча с клиентом. Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |
