# 📖 Ситуация

В компании **RepairFlow** используется настольное приложение для регистрации и обработки заявок на ремонт оборудования.

Приложение позволяет специалистам службы эксплуатации и IT-отдела:

* создавать новые заявки на ремонт аппаратного и программного обеспечения;
* отслеживать статус выполнения ремонтов;
* управлять жизненным циклом заявок через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать наименования оборудования и описания неисправностей с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации заявки

Форма используется для создания нового обращения на ремонт оборудования.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Equipment

Назначение:

ввод наименования оборудования или описания неисправности.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле Technician

Назначение:

имя техника, оформившего заявку.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список Category

Назначение:

выбор категории неисправности.

Возможные значения:

* Hardware
* Software
* Network

---

## Список Priority

Назначение:

выбор приоритета обработки заявки.

Возможные значения:

* Low
* Medium
* High

# Шаг 2. Генерация наименований оборудования через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать названия оборудования и описания неисправностей**, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 наименований оборудования с указанием типа неисправности для службы ремонта. Распредели по категориям: 4 Hardware (серверы, рабочие станции, принтеры, ноутбуки), 3 Software (ОС, антивирус, драйверы), 3 Network (маршрутизаторы, коммутаторы, точки доступа). Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |

---

## Пример ответа GPT

```
Сервер Rack A: отказ блока питания
Рабочая станция Dell: ошибка загрузки ОС
Принтер LaserJet: замятие бумаги и сбой картриджа
Ноутбук HP: перегрев и отключение вентилятора
ОС Windows 11: критическое обновление не применяется
Антивирус Kaspersky: конфликт с драйверами видеокарты
Обновление драйверов чипсета AMD
Маршрутизатор Cisco: потеря пакетов на WAN-порту
Коммутатор HP: сбой PoE на портах 5-8
Точка доступа Ubiquiti: нестабильный сигнал 5 ГГц
```
