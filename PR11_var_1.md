# 📖 Ситуация

В компании **TechSupport** используется настольное приложение для регистрации и обработки инцидентов службы технической поддержки.

Приложение позволяет сотрудникам и специалистам IT-отдела:

* создавать новые заявки на техническую поддержку;
* отслеживать статус их решения;
* управлять жизненным циклом инцидентов через табличный интерфейс.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы (RDP);
* внутри защищённых виртуальных сред (VDI);
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* находит элементы по их визуальному шаблону;
* выполняет действия по координатам найденных элементов.

Дополнительно в данной работе используется **интеграция с искусственным интеллектом (LLM)**.
Робот будет **генерировать названия инцидентов с помощью GPT**, а затем автоматически добавлять их в систему.

В рамках практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который:

* взаимодействует с приложением через **Machine Vision**;
* использует **GPT для генерации данных**.

# Форма регистрации заявки

Форма используется для создания нового обращения в службу поддержки.

Пользователь вводит данные и добавляет запись в таблицу.

---

## Поле Issue Title

Назначение:

ввод темы инцидента.

В данной работе значение **генерируется автоматически с помощью GPT**.

---

## Поле User

Назначение:

имя сотрудника, сообщившего о проблеме.

Примеры:

* Ivan Petrov
* Anna Smirnova

---

## Список System

Назначение:

выбор системы, в которой возникла проблема.

Возможные значения:

* CRM
* ERP
* Email

---

## Список Priority

Назначение:

выбор приоритета обработки заявки.

Возможные значения:

* Low
* Medium
* High

---

# Таблица заявок

Структура таблицы:

| ID | Title | User | System | Priority | Status |

---

# Возможные статусы

```
New
Closed
```

---

# Кнопки управления заявками

## Resolve Issue

Изменяет статус выбранной заявки на

```
Closed
```

Кнопка неактивна, если статус заявки уже `"Closed"` или строка не выбрана.

---

## Assign User

Назначает ответственного специалиста на выбранную заявку.

Кнопка активна только при выборе строки в таблице.

# Шаг 2. Генерация названий инцидентов через GPT

## Зачем выполняется шаг

Робот должен **самостоятельно придумать темы инцидентов**, используя искусственный интеллект.

Это демонстрирует интеграцию **LLM в RPA-процесс**.

---

## Что необходимо сделать

Использовать блок:

**AI → Выполнить запрос в GPT**

### Параметры блока

| Параметр                   | Значение                                                                                                                                                                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Выполнить запрос к         | PuzzleGPT                                                                                                                                                                                                                                                                                            |
| Запрос                     | Сгенерируй 10 названий IT-инцидентов для службы поддержки. Темы: ошибка доступа к CRM, сбой авторизации в ERP, проблемы синхронизации почты, таймаут API, падение сервера, нехватка дискового пространства, задержка сети, ошибка загрузки данных, сбой подключения к БД, обновление сертификатов. Верни только список названий без номеров, каждое название с новой строки. |
| Максимальное число токенов | 500                                                                                                                                                                                                                                                                                                  |
| Уровень случайности        | 0.5                                                                                                                                                                                                                                                                                                  |

---

## Пример ответа GPT

```
Ошибка доступа к модулю отчётности CRM
Сбой авторизации в производственной ERP
Проблема синхронизации входящей почты
Таймаут при вызове внешнего API
Критическое падение веб-сервера
Нехватка дискового пространства на хранилище
Высокая задержка корпоративной сети
Ошибка пакетной загрузки данных в БД
Сбой подключения к резервному кластеру
Истечение срока действия SSL-сертификата
```
